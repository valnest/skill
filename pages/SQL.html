SQL — применяется для создания, модификации и управления данными в реляционной базе данных, управляемой соответствующей системой управления базами данных.
В базовом варианте SQL является информационно-логическим языком, а не языком программирования.

Система управления базами данных (СУБД) — совокупность программных и лингвистических средств общего или специального назначения, обеспечивающих управление созданием и использованием баз данных.

База данных (БД) - под базой данных (БД) понимают хранилище структурированных данных, при этом данные должны быть непротиворечивы, минимально избыточны и целостны.

Реляционная база данных — база данных, основанная на реляционной модели данных.
Реляционные БД представляют связанную между собой совокупность таблиц-сущностей базы данных.
Каждая таблица БД представляется как совокупность строк и столбцов, где строки соответствуют экземпляру объекта, конкретному событию или явлению, а столбцы - атрибутам (признакам, характеристикам, параметрам) объекта, события, явления.
При практической разработке БД таблицы-сущности зовутся таблицами, строки-экземпляры - записями, столбцы-атрибуты - полями.

Преимущества SQL - наличие стандартов и независимость от конкретной СУБД.
Недостатки SQL - сложность.

Операторы определения данных (Data Definition Language, DDL) 
  CREATE - создает объект БД (саму базу, таблицу, представление, пользователя и т. д.)
  ALTER - изменяет объект
  DROP - удаляет объект
Операторы манипуляции данными (Data Manipulation Language, DML)
SELECT 
  SELECT - выбирает данные, удовлетворяющие заданным условиям, SELECT * FROM table
  
    SELECT DISTINCT speed, ram
    FROM PC
    WHERE price < 500
    ORDER BY ram DESC;
  
    SELECT model, speed, hd
    FROM PC
    WHERE hd IN (10, 20) AND
    model IN (SELECT model
    FROM product
    WHERE maker = 'A');
  
    SELECT model, speed
    FROM PC
    WHERE price BETWEEN 400 AND 600;
  
    SELECT *
    FROM Ships
    WHERE class NOT LIKE '%go' AND
    class LIKE '%o' ;
    select * from movies where studio_id = (select id from studios where studio_name like '%Pictures');
    select * from movies where studio_id = ANY (select id from studios where studio_name like '%Pictures');
    select * from movies where studio_id IN (select id from studios where studio_name like '%Pictures');
    select distinct country from (select title, country, revenue from movies order by revenue desclimit 5) AS top_movies;
    select country, studio_id, max(oscar_winner) from movies
    Group by country, studio_id
    order by 2 desc
    select country, max(oscar_winner) from movies
    Group by country
    HAVING MAX(oscar_winner) > 1
    order by 2 desc
  
  В выражении SELECT мы можем вводить подзапросы четырьмя способами:
    В условии в выражении WHERE
    В качестве таблицы для выборки в выражении FROM
    В качестве спецификации столбца в выражении SELECT
    В условии в выражении HAVING

  Агрегатные функции:
  COUNT(*) - Возвращает количество строк источника записей
      select count(*) from movies
      select count(rating) from movies
  SUM - Возвращает сумму значений в указанном столбце
      select sum(revenue) from movies
  AVG - Возвращает среднее значение в указанном столбце
      select avg(budget) from movies
  MIN - Возвращает минимальное значение в указанном столбце
  MAX - Возвращает максимальное значение в указанном столбц
      select max(oscar_winner) from movies
JOIN 
Неявное соединение:
  select * from persons, staff where persons.id = staff.person_id
  либо
  select * from persons CROSS JOIN staff where persons.id = staff.person_id

Явное соединение:
  select [columns] from table1 [INNER] | {[LEFT] | [RIGHT] | [FULL]} [OUTER] JOIN table2
  ON table1.[column] = table2.[column]
  
  select * from persons [INNER] JOIN staff ON persons.id = staff.person_id
  
  select [columns] from table1 t1 [LEFT] | [RIGHT] | [FULL] JOIN table2 t2
  ON t1.[column] = t2.[column]


  select * from persons p JOIN staff s
  ON p.id = s.person_id
  
  select * from persons p LEFT JOIN staff s
  ON p.id = s.person_id
  
  select * from persons p RIGHT JOIN staff s
  ON p.id = s.person_id
  
  select * from persons p FULL JOIN staff s
  ON p.id = s.person_id
UNION 
UNION – объединение:                                         
  UNION применяется для объединения результатов двух SQL-запросов
  в единую таблицу, состоящую из схожих строк. Оба запроса должны
  возвращать одинаковое число столбцов и совместимые типы данных
  в соответствующих столбцах!!!

  select [columns] from table1
  UNION [ALL]
  elect [columns] from table2
INTERSECT 
  INTERSECT - Пересечение:
  Пересечение — множество, состоящее из элементов, которые
  одновременно принадлежат всем данным множествам.
  INTERSECT, также как и UNION обладает ограничением на результат
  запросов, которые должны быть совместимы по объединению

  select [columns] from table1
  INTERSECT
  select [columns] from table2
MINUS или EXCEPT 

  MINUS или EXCEPT - Разность:
  Разность двух множеств — это операция, результатом которой
  является множество, в которое входят все элементы первого
  множества, не входящие во второе множество

  select [columns] from table1
  MINUS | EXCEPT
  select [columns] from table2

INSERT 
  INSERT добавляет новые данные

  INSERT INTO table ([columns])
  VALUES ([values])
  INSERT INTO persons (id, full_name, birth_date, gender)
  VALUES (1000, 'Chris Pratt', '1979-06-21', 'male')
UPDATE 
  UPDATE изменяет существующие данные

  UPDATE table SET [column1] = [value],
  [column2] = [value]
  WHERE ...
  UPDATE persons SET full_name = 'Chris Hemsworth',
  Birth_date = '1983-08-11' WHERE id = 1000
DELETE 
  DELETE удаляет данные

  DELETE FROM table WHERE ...
  DELETE from persons WHERE id = 1000
Операторы определения доступа к данным (Data Control Language, DCL) 
  GRANT - предоставляет пользователю (группе) разрешения на определенные операции с объектом
  REVOKE - отзывает ранее выданные разрешения
  DENY - задает запрет, имеющий приоритет над разрешением
Операторы управления транзакциями (Transaction Control Language, TCL) 
  COMMIT - применяет транзакцию
  ROLLBACK - откатывает все изменения, сделанные в контексте текущей транзакции
  SAVEPOINT - делит транзакцию на более мелкие участки

  START TRANSACTION/COMMIT/ROLLBACK:
  SET AUTOCOMMIT = [1|0] | [ON|OFF]
  START TRANSACTION;
  COMMIT;
  ROLLBACK;
